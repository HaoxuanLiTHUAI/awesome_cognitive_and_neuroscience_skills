# Contributing to Awesome Cognitive and Neuroscience Skills

Thank you for your interest in contributing! This project relies on community participation to ensure the quality and accuracy of its domain knowledge.

## Important Context

All existing skills in this repository were **generated by AI** from academic papers and publicly available sources. While automated checks were applied during generation, the content has **not been individually verified by human domain experts**. Community verification is the most valuable contribution you can make.

---

## Types of Contributions

### 1. Verify an Existing Skill (Most Needed)

This is the **highest-impact contribution** you can make. Pick any skill with `review_status: "ai-generated"` and verify its content:

- **Check citations**: Do the cited authors, years, and findings actually exist? Look up the papers.
- **Check parameters**: Are numerical thresholds, filter settings, timing windows, etc. accurate?
- **Check methodology**: Is the recommended pipeline/workflow correct and current?
- **Report findings**: Open an issue for errors, or submit a PR with corrections.

When you verify a skill, update its `review_status` frontmatter to `"community-reviewed"` and add yourself to the `authors` list with a `(reviewer)` annotation.

### 2. Improve an Existing Skill

- Add missing decision logic or edge cases
- Update outdated recommendations with newer references
- Improve clarity or correct logical flow
- Add content to `references/` subdirectory

### 3. Submit a New Skill

Before writing, check that your skill encodes genuine domain knowledge (see the Litmus Test below). Then:

1. Fork the repository
2. Create a skill directory under `skills/` using **kebab-case** naming
3. Write a `SKILL.md` following the structure below
4. Submit a PR

### 4. Report an Error

If you spot an incorrect citation, wrong parameter, or methodological error, [open an issue](https://github.com/HaoxuanLiTHUAI/awesome_cognitive_and_neuroscience_skills/issues) using the "Citation/Parameter Error" template.

### 5. Share a Usage Case (No Git Required)

Used a skill in your research? Share your experience directly from Claude Code:

1. After using any skill, say **"share this case"** (or "分享这个案例")
2. The `share-case` meta-skill guides you through a quick survey and context extraction
3. You review and control exactly what gets shared
4. The case is submitted to [GitHub Discussions > Show & Tell](https://github.com/HaoxuanLiTHUAI/awesome_cognitive_and_neuroscience_skills/discussions/categories/show-tell)

### 6. Contribute a New Skill (No Git Required)

Have domain expertise to share? Generate and submit a skill from your terminal:

1. Say **"contribute a skill"** (or "把这个方法做成 skill")
2. The `contribute-skill` meta-skill helps you generate a properly formatted SKILL.md
3. You review the generated skill and approve before submission
4. The skill is submitted as a [GitHub Issue](https://github.com/HaoxuanLiTHUAI/awesome_cognitive_and_neuroscience_skills/issues) for maintainer review

### 7. Verify a Skill Interactively (No Git Required)

Domain expert? Help verify AI-generated skills through structured review:

1. Say **"verify a skill"** (or "验证这个 skill")
2. The `verify-skill` meta-skill walks you through a 5-phase interactive verification
3. You assess parameters, test against realistic scenarios, and provide expert feedback
4. The verification report is submitted to [GitHub Discussions > Verification](https://github.com/HaoxuanLiTHUAI/awesome_cognitive_and_neuroscience_skills/discussions/categories/verification)

> **One-time setup for methods 5-7:**
>
> ```bash
> sudo apt install gh   # Ubuntu/Debian — other OS see https://cli.github.com/
> gh auth login
> ```
>
> That's it. If `gh` is not available, the meta-skills will save your contribution as a local file instead.

## Skill Writing Requirements

### YAML Frontmatter (Required)

```yaml
---
name: "Human-Readable Skill Name"
description: "One-sentence summary of the domain knowledge encoded"
review_status: "ai-generated"  # or "community-reviewed" / "expert-verified"
---
```

`name`, `description`, and `review_status` are mandatory.

### Required Sections

Every SKILL.md must include, in order:

1. **Purpose** — What domain knowledge this skill encodes
2. **When to Use** — Trigger conditions for the skill
3. **Research Planning Protocol** — Standard preamble (copy from any existing skill)
4. **Verification Notice** — AI-generation disclaimer (copy from any existing skill)
5. **Domain-specific content** — The actual skill logic

### Key Rules

- **Under 500 lines** — Move supplementary material to `references/`
- **Cite every number** — All numerical parameters need `(Author, Year)` citations
- **No page numbers** — Use `(Luck, 2014)` not `(Luck, 2014, p. 187)` — page numbers are high-risk for AI hallucination
- **kebab-case directories** — e.g., `eeg-artifact-rejection`

### The Litmus Test

> Would a competent programmer who has never taken a cognitive science course get this wrong?

If **yes** → it is domain knowledge and belongs in a skill.
If **no** → it is general knowledge and does NOT belong.

## Review Status Promotion Path

Skills progress through three verification levels:

| Status | Meaning | Requirements to Achieve |
|---|---|---|
| `ai-generated` | Initial state | Generated by AI, no human verification |
| `community-reviewed` | Community-verified | At least 1 contributor has verified citations and parameters; PR approved by a maintainer |
| `expert-verified` | Expert-confirmed | A researcher with domain expertise has reviewed methodological accuracy; documented in PR |

### How to Promote a Skill

1. **ai-generated → community-reviewed**: Submit a PR that documents which citations and parameters you verified, with links to source papers or textbooks.
2. **community-reviewed → expert-verified**: A domain expert (PhD student or above in the relevant field) reviews the skill and confirms methodological accuracy. The reviewer should be identified in the PR.

## Pull Request Checklist

Before submitting a PR, ensure:

- [ ] SKILL.md has required YAML frontmatter (`name`, `description`, `review_status`)
- [ ] Includes Research Planning Protocol section
- [ ] Includes Verification Notice section
- [ ] All numerical parameters have `(Author, Year)` citations
- [ ] Citations do not contain specific page numbers
- [ ] SKILL.md is under 500 lines
- [ ] Directory name uses kebab-case
- [ ] Commit messages use conventional prefixes (`feat:`, `fix:`, `docs:`, etc.)
